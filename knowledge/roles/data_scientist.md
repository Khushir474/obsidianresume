ATS RULES (non-negotiable):

  "statistical modeling," use those exact words, not synonyms
- Spell out every acronym on first use: "Retrieval-Augmented

  "Large Language Models (LLMs)" — abbreviate after
- Every hard keyword from List A must appear in BOTH the Skills
  section AND at least one Work Experience bullet
- Every soft keyword from List B must be woven naturally into
  at least one bullet
- Dates must be written as: "June 2025 – Present" — no apostrophes,
  no abbreviations
- Single-column format only — no tables, no text boxes, no icons

ROLE-SPECIFIC ATS SIGNALS TO HIT:
- Experimentation: A/B test design, hypothesis testing, statistical significance, p-value, confidence interval
- Modeling: feature engineering, model selection, cross-validation, precision/recall, AUC, bias-variance tradeoff
- Stack: Python, scikit-learn, PyTorch, SQL, Databricks, Spark, Snowflake, pandas, NumPy — use whichever the JD names
- Business: revenue impact, cost savings, efficiency gains, churn, conversion, retention

EXTRAPOLATION:
Fill the gap between what I wrote and what a strong data scientist
would have done. If I built a pipeline, assume I handled data quality
checks, versioning, and output monitoring. If I ran analysis, assume
I pressure-tested with statistical validation. If I ran an experiment,
assume the full arc: defined hypothesis, chose metric, set significance
threshold, documented results, recommended action. If I built a model,
assume I iterated on feature engineering, compared baselines, and
tracked accuracy/precision/recall improvements.
Push to what I can defend in an interview.

WRITING STYLE — Paul Graham (human layer, applied after ATS):
- Use the simplest accurate verb: "cut" not "reduced," "built" not
  "developed and implemented"
- Put the result first, method second
- One strong idea per bullet — never stack three claims
- No: leveraged, spearheaded, orchestrated, utilized, streamlined
- No passive voice
- No filler endings — if it's not a number, cut it
